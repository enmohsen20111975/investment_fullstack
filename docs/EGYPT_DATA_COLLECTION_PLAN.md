# خطة جمع بيانات مصر فقط (الأسهم + الذهب + الأخبار)

## الهدف
بناء Pipeline سحب بيانات يركز على **مصر فقط** من:
1. Investing (قسم مصر)
2. Mubasher (مصر)

ويغطي:
- جميع صفحات الأسهم المصرية (EGX)
- بيانات الذهب/السلع المرتبطة بالسوق المصري
- الأخبار الاقتصادية والمالية الخاصة بمصر
- صفحات التفاصيل لكل سهم/خبر (وليس الاكتفاء بصفحة الفهرس)
- كل الجداول المالية والتاريخية المتاحة في الصفحات
- كل بيانات الرسوم البيانية (Chart Series) وتحويلها لسلاسل زمنية قابلة للتحليل

---

## النطاق (Egypt Only)
- الدولة المستهدفة: مصر فقط
- السوق المستهدف: EGX فقط
- اللغة: عربي/إنجليزي حسب الصفحة مع توحيد المخرجات
- الاستبعاد: أي أسواق غير مصرية يتم تجاهلها أثناء الاستخراج

---

## مخرجات البيانات المطلوبة

### 1) الأسهم (Stocks)
- رمز السهم (Ticker)
- اسم الشركة
- آخر سعر
- التغير (قيمة + نسبة)
- سعر الفتح / الأعلى / الأدنى
- حجم التداول / قيمة التداول
- وقت آخر تحديث
- القطاع (إن وجد)
- رابط صفحة تفاصيل السهم

### 2) الذهب والسلع (Gold/Commodities)
- اسم الأداة (ذهب فوري/عقود/عيارات محلية إن توفرت)
- السعر الحالي
- التغير (قيمة + نسبة)
- نطاق اليوم (High/Low)
- وقت التحديث
- رابط التفاصيل

### 3) الأخبار (News)
- عنوان الخبر
- ملخص قصير
- التصنيف (أسهم/اقتصاد/ذهب/بنوك...)
- تاريخ ووقت النشر
- اسم المصدر/الكاتب
- رابط الخبر
- محتوى الخبر الكامل
- الوسوم (Tags) إن وجدت

### 4) تفاصيل إضافية لكل سهم (Deep Details)
- نبذة الشركة
- مضاعفات وتقييمات أساسية (إذا متاحة)
- بيانات تاريخية للأسعار
- توزيعات الأرباح
- الأحداث الجوهرية / الإفصاحات

### 5) الجداول والرسوم البيانية (Tables & Charts) - أولوية قصوى
- استخراج كامل لأي جدول ظاهر في صفحات الأسهم/الذهب/الأخبار (وليس صفوف أولية فقط)
- التقاط أسماء الأعمدة كما هي + ترتيبها + نوع كل عمود
- حفظ كل صف في الجداول مع رقم الصفحة + وقت السحب
- سحب بيانات الرسوم البيانية كسلسلة زمنية (timestamp, open, high, low, close, volume) عند توفرها
- حفظ إعدادات الرسم المهمة: الإطار الزمني (1D, 1W, 1M, 1Y...) ونوع الرسم
- ربط كل نقطة زمنية بمصدرها (الرابط + اسم الصفحة + الأداة)

### 6) التحليلات والتوقعات (Analysis & Forecasts)
- جمع أي تحليل منشور مرتبط بمصر: ملخص التحليل، الاتجاه (صاعد/هابط/محايد)، الأفق الزمني
- جمع أي سعر/هدف متوقع أو نطاق سعري متوقع مع تاريخ الإصدار
- استخراج التوصيات (شراء/احتفاظ/بيع) عند توفرها
- ربط كل تحليل/توقع بالمصدر الأصلي + تاريخ النشر + اسم المحلل/الجهة إن وجد
- إنشاء توقعات داخلية إضافية من البيانات التاريخية (Baseline Forecast) مع درجة ثقة

## معمارية التنفيذ

## المرحلة A: اكتشاف الروابط (Discovery)
الهدف: الحصول على كل روابط الصفحات المصرية المطلوبة قبل السحب التفصيلي.

1. نقطة دخول Investing مصر
   - استخراج روابط: الأسهم المصرية، السلع/الذهب، الأخبار المصرية.
2. نقطة دخول Mubasher مصر
   - استخراج روابط: شاشات السوق المصري، الأسهم، الأخبار، صفحات التفاصيل.
3. بناء قائمة URLs أولية مع وسم كل رابط:
   - source: investing | mubasher
   - type: list | detail
   - domain_scope: egypt

مخرج المرحلة:
- ملف روابط خام `discovered_urls_egypt.json`

---

## المرحلة B: جمع القوائم كاملة (Pagination Crawl)
الهدف: عدم التوقف عند الصفحة الأولى.

1. لكل صفحة قائمة:
   - كشف نمط الـ pagination (page, p, start, next link).
   - التكرار حتى آخر صفحة متاحة.
2. جمع روابط التفاصيل من كل صفحة قائمة.
3. إزالة التكرار (Dedup) بالاعتماد على الرابط الأساسي + معرف داخلي.
4. حفظ checkpoint بعد كل N صفحة لضمان الاستئناف عند التوقف.

مخرج المرحلة:
- `stocks_list_egypt.jsonl`
- `gold_list_egypt.jsonl`
- `news_list_egypt.jsonl`
- `detail_urls_egypt.json`

---

## المرحلة C: سحب صفحات التفاصيل (Detail Scrape)
الهدف: استخراج كل البيانات الممكنة من صفحات السهم/الذهب/الخبر.

1. فتح كل رابط تفصيلي في طابور مهام (Queue).
2. تطبيق Extractors حسب النوع:
   - StockDetailExtractor
   - GoldDetailExtractor
   - NewsDetailExtractor
3. توحيد الحقول في Schema واحدة.
4. حفظ كل سجل مع:
   - source_url
   - fetched_at
   - parser_version
5. استخراج الجداول من HTML مباشرة مع الحفاظ على البنية:
   - table_name
   - columns[]
   - rows[]
6. استخراج بيانات الرسوم البيانية من:
   - JSON المضمّن داخل الصفحة
   - أو XHR/Fetch endpoints الخاصة بالشارت أثناء التحميل
7. توحيد السلاسل الزمنية في schema موحّد:
   - instrument_id, interval, timestamp, open, high, low, close, volume, source

مخرج المرحلة:
- `stocks_detail_egypt.jsonl`
- `gold_detail_egypt.jsonl`
- `news_detail_egypt.jsonl`
- `egypt_tables_raw.jsonl`
- `egypt_chart_series_raw.jsonl`

---

## المرحلة D: التنظيف والتوحيد (Normalize)
1. توحيد صيغ الأرقام العربية/الإنجليزية.
2. تحويل كل التواريخ إلى UTC + حفظ timezone الأصلي.
3. توحيد العملة (EGP/USD) مع حقل `currency`.
4. إزالة أي سجل خارج نطاق مصر.
5. إنشاء `market=EGX` بشكل إلزامي للأسهم المصرية.
6. توحيد توصيف التوقعات في قاموس موحد:
   - recommendation: buy | hold | sell | neutral
   - horizon: intraday | weekly | monthly | quarterly | yearly
   - confidence_score: 0..1

مخرج المرحلة:
- `egypt_market_dataset_clean.jsonl`
- `egypt_analysis_forecasts_clean.jsonl`

---

## المرحلة F: طبقة التحليل والتوقعات (Analytics Layer)
الهدف: إنتاج تحليلات قابلة للاستخدام بجانب البيانات الخام.

1. تحليل اتجاه السهم/الذهب من السلاسل الزمنية:
   - العائد اليومي/الأسبوعي/الشهري
   - الزخم (Momentum)
   - التذبذب (Volatility)
2. توقعات قصيرة ومتوسطة الأجل لكل أداة:
   - Baseline: Moving Average / Exponential Smoothing
   - حفظ نقطة التوقع + نطاق ثقة
3. ربط الأخبار بالأسعار:
   - قياس تأثير زمني بعد نشر الخبر (Event Impact Window)
4. تقييم دقة التوقعات دوريًا:
   - MAE, MAPE, RMSE

مخرج المرحلة:
- `egypt_analytics_daily.jsonl`
- `egypt_forecasts.jsonl`
- `egypt_forecast_backtest.jsonl`

## التعامل مع القيود (Login / Anti-Bot)

### Mubasher
- صفحة الدخول ظهرت كمتطلب لبعض المسارات.
- التنفيذ المقترح:
  1. البدء بمسارات عامة بدون تسجيل دخول.
  2. إذا البيانات المهمة خلف تسجيل الدخول:
     - استخدام حساب خدمة مخصص + Session cookies.
     - تخزين آمن للبيانات الحساسة في `.env`.

### Investing
- تم رصد redirect/anti-bot على بعض الطلبات.
- التنفيذ المقترح:
  1. استخدام Playwright للرندر الحقيقي.
  2. User-Agent واقعي + pacing للطلبات.
  3. Retry بسياسة backoff وتدوير محدود.

## جودة البيانات (Quality Gates)
1. التغطية:
   - نسبة روابط التفاصيل المسحوبة >= 95%
   - نسبة الجداول المستخرجة مقارنة بالجداول المرصودة >= 98%
   - نسبة الرسوم البيانية المحوّلة لسلاسل زمنية >= 95%
2. الاكتمال:
   - الأخبار بدون عنوان = 0%
   - الأسهم بدون ticker < 1%
   - صفوف الجداول بدون أعمدة مرجعية = 0%
3. الصحة:
   - رفض أي سجل سعره غير رقمي
   - رفض أي نقطة شارت بدون timestamp صالح
4. الحداثة:
   - تحديثات السوق ضمن نافذة زمنية مقبولة
5. الاتساق:
   - توافق عدد نقاط السلاسل الزمنية مع الإطار الزمني المطلوب لكل أداة
6. جودة التحليل والتوقع:
   - وجود مصدر واضح لكل تحليل خارجي بنسبة 100%
   - تغطية التوقعات للأدوات الرئيسية المصرية >= 90%
   - حد أقصى لـ MAPE يتم مراجعته شهريًا وتحسينه تدريجيًا

## الجدول التنفيذي المقترح
- Job سريع كل 15 دقيقة: أسعار الأسهم والذهب
- Job كل ساعة: أخبار جديدة
- Job يومي ليلاً: تحديث عميق (تفاصيل + تاريخي)
- Job يومي بعد إغلاق السوق: تشغيل التحليل والتوقع وتحديث مقاييس الدقة
