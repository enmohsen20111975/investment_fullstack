# وثيقة متطلبات المنتج (PRD) — منصة استثمار البورصة المصرية

## 1) الهدف
تحويل بيانات السهم التفصيلية الظاهرة في واجهة المستخدم إلى **تحليل عميق** و**نصيحة استثمارية متقدمة** قابلة للتنفيذ، مع الاعتماد على بيانات حقيقية فقط، وتقديم مخرجات واضحة للويب والموبايل.

## 2) سياق المنتج
المنصة الحالية تعرض:
- لوحة تحكم (إجمالي الأسهم، الحلال، المشكوك، الحرام)
- جداول أعلى الرابحين/الخاسرين
- صفحة تفاصيل سهم (سعر لحظي، امتثال شرعي، مقاييس مالية، مؤشرات فنية، رسم سعري)

المطلوب الآن: استخدام هذه البيانات كأساس لمحرك قرار استثماري أعمق يقدم توصيات عملية آمنة ومتقدمة.

## 3) نطاق هذا الإصدار
### ضمن النطاق
1. بناء تحليل سهم متقدم يعتمد على:
   - بيانات الأسعار والتاريخ
   - المقاييس الأساسية (PE, PB, EPS, ROE, Debt/Equity, Dividend Yield)
   - المؤشرات الفنية (MA50, MA200, RSI, دعم/مقاومة)
   - إشارات المصادر المفتوحة المتاحة فعليًا
2. توليد نصيحة استثمارية قابلة للتنفيذ تشمل:
   - سيناريو محافظ (Saving)
   - سيناريو نمو (Growth)
   - نقاط دخول/وقف خسارة/مستويات مراقبة
3. تخزين مخرجات التحليل في قاعدة البيانات مع التاريخ والوقت لاستخدامها كمصدر واجهة.
4. توحيد مخرجات API بحيث تناسب واجهة الويب وتطبيق الموبايل.

### خارج النطاق (حاليًا)
- تنفيذ أوامر شراء/بيع حقيقية لدى وسيط تداول.
- التنبؤ المؤكد بالعوائد (أي ضمان ربح).

## 4) المستخدمون المستهدفون
1. مستثمر مبتدئ يحتاج إرشاد واضح وآمن.
2. مستثمر متوسط يبحث عن مزيج حفظ رأس مال + نمو.
3. واجهات عميلة (Frontend/Mobile) تحتاج بيانات جاهزة للعرض دون معالجة معقدة.

## 5) متطلبات اللغة وتجربة الاستخدام
- اللغة الأساسية للتطبيق: العربية.
- المصطلحات الفنية المعيارية يمكن أن تبقى بالإنجليزية (RSI, MA, PE...) عند الحاجة.
- جميع الرسائل التفسيرية والتوصيات والإشعارات بالعربية.

## 6) متطلبات البيانات (من الصورة المرفقة + النظام الحالي)
يجب أن يتعامل المحرك مع الحقول التالية كحد أدنى:
1. معلومات عامة: ticker, الاسم, القطاع
2. السعر: current_price, previous_close, open, high, low
3. الحركة: نسبة/قيمة التغير
4. السيولة: volume
5. الأساسيات: market_cap, PE, PB, EPS, ROE, debt_to_equity, dividend_yield
6. الفنيات: MA50, MA200, RSI, support, resistance
7. الامتثال الشرعي: compliance_status + compliance_note
8. التاريخ السعري: OHLCV لفترة زمنية قابلة للاختيار
9. إشارات المصادر الخارجية: Yahoo/TradingView/MarketWatch + توضيح unavailable للمصادر المحجوبة

## 7) متطلبات التحليل العميق
### 7.1 محرك تقييم السهم (Scoring)
- إنتاج درجة مركبة 0-100 مع تفصيل المكونات:
  - جودة أساسية (Fundamental Score)
  - اتجاه فني (Technical Score)
  - مخاطرة (Risk Score)
  - موثوقية البيانات (Data Availability Score)

### 7.2 مقاييس المخاطر والعائد
- 7D Return
- 30D Return
- Annualized Volatility
- Max Drawdown
- Trend Signal
- Risk Level (low/medium/high)

### 7.3 محرك النصيحة المتقدمة
إخراج توصية قابلة للتنفيذ:
- الحالة: شراء / احتفاظ / تقليل / تجنب
- الثقة: منخفضة/متوسطة/مرتفعة
- أسباب القرار (3-5 أسباب)
- مخاطر رئيسية (2-4 مخاطر)
- قواعد حماية (وقف خسارة، مراجعة دورية، حدود تركّز)

## 8) متطلبات النصيحة المتقدمة (Advanced Advice)
### 8.1 Advice للمدخر (Saving)
- يفضّل أسهم منخفضة المخاطر + توزيعات أرباح + مديونية أقل.
- يحدّد نسبة من رأس المال لهذا المسار.

### 8.2 Advice للنمو (Growth)
- يركز على الاتجاه الفني والزخم والنمو المتوقع مع مخاطرة محسوبة.
- يحدّد نسبة من رأس المال لهذا المسار.

### 8.3 Advice لإدارة رأس المال
- توزيع مقترح بين: نقد / ذهب / فضة / أسهم.
- داخل الأسهم: تقسيم Saving vs Growth.
- تنبيهات إعادة موازنة شهرية أو عند انحراف > 5%.

## 9) متطلبات API
### 9.1 Endpoint تحليل عميق
- Endpoint قائم: `/api/stocks/{ticker}/deep-insights`
- المطلوب:
  - الاستمرار في إرجاع `real_data_only=true`
  - إرجاع `decision_metrics` + مصادر البيانات + حالة كل مصدر
  - حفظ snapshot في DB لكل تشغيل

### 9.2 Endpoint تاريخ التحليل
- Endpoint قائم: `/api/stocks/{ticker}/deep-insights/history`
- المطلوب:
  - دعم limit
  - إرجاع أحدث snapshots مع `run_id`, `fetched_at`

### 9.3 Endpoint نصيحة متقدمة
- يوصى بإضافة endpoint مخصص (إذا لم يكن كافيًا في endpoint التوصية الحالي):
  - `/api/stocks/{ticker}/advanced-advice`
- يعيد:
  - recommendation summary
  - saving plan
  - growth plan
  - safety rules
  - confidence + data quality

## 10) متطلبات التخزين (DB)
البيانات يجب أن تحفظ مع وقت الإنشاء:
1. `stock_source_snapshots` — لكل مصدر على حدة
2. `stock_deep_insight_snapshots` — ملخص التحليل المركب

متطلبات إضافية:
- الاحتفاظ بآخر N snapshots لكل سهم (سياسة retention)
- دعم استرجاع سريع لأحدث snapshot للواجهة

## 11) متطلبات الواجهة (من واقع الصور)
1. نافذة تفاصيل السهم يجب أن تعمل Popup/Subscreen واضح على كل الشاشات.
2. في الموبايل: Fullscreen + scrolling داخلي كامل.
3. قسم التوصية المتقدمة داخل تفاصيل السهم يجب أن يظهر:
   - ملخص القرار
   - مستوى المخاطرة
   - أسباب/مخاطر
   - خطة Saving vs Growth

## 12) متطلبات الاعتمادية والجودة
1. حلقة اختبار تلقائية لجودة المصادر (Loop Test) قبل الاعتماد.
2. تعريف واضح لـ fallback behavior:
   - إذا مصدر محجوب (403) => `blocked=true`
   - إذا رمز غير مدعوم (404) => `unsupported=true`
   - لا يتم اختراع بيانات بديلة وهمية.
3. تقليل ضوضاء السجلات مع الحفاظ على trace مفيد.

## 13) مؤشرات النجاح (KPIs)
1. اكتمال بيانات التحليل لكل سهم (Data Completeness) ≥ 70% من الحقول الأساسية.
2. استقرار المصادر المتاحة (Yahoo/TradingView/MarketWatch) ضمن بيئة التشغيل الحالية.
3. زمن استجابة endpoint التحليل العميق ضمن الحدود المقبولة لتجربة المستخدم.
4. زيادة استخدام صفحة التفاصيل والتوصيات المتقدمة في الواجهة.

## 14) المخاطر والقيود
1. بعض المصادر تمنع scraping المباشر (403/anti-bot).
2. بعض الرموز غير مدعومة لدى مزود معين (404).
3. مفاتيح AI غير صالحة/غير مهيأة قد توقف جزء AI (يجب graceful degradation).

## 15) خطة تنفيذ موجزة
1. Phase 1: تثبيت جودة البيانات + snapshots
2. Phase 2: محرك تقييم مركب + advanced advice
3. Phase 3: تحسين عرض النصيحة في الواجهة (Desktop/Mobile)
4. Phase 4: تحسينات الأداء والمراقبة

## 16) ملاحظات تسليم
- هذه الوثيقة تمثل مرجع المنتج الرسمي للتحليل العميق والنصيحة المتقدمة.
- أي AI أو مطور جديد يجب البدء بها قبل تنفيذ ميزات إضافية.
